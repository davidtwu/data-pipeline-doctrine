# Product Context

## Overview
A personal technical blog for a principal engineer focused on data engineering, ETL pipelines, and the intersection of AI/LLMs with data infrastructure. The blog serves as a media leverage tool to build internal and external influence.

## Target Audience
- Senior and staff+ data engineers at large tech companies
- Engineering leaders evaluating data platform architecture decisions
- ML engineers working on the LLM + data pipeline intersection

## Core Philosophy
- **Write in Obsidian, publish with git push.** The authoring experience lives in Obsidian; the blog is a rendering layer.
- **Ship over polish.** The system must make publishing a 300-word TIL post feel trivially easy.
- **Zero maintenance.** No databases, no servers, no CMS logins. Static files on GitHub Pages.
- **Content is king.** The site should look clean and professional but never require fiddling with design. All energy goes to writing.

## Content Types
1. **TIL (Today I Learned)** — 200-500 word micro-posts. Published 1-2x per week.
2. **Deep Dives** — 1,500-3,000 word technical essays. Published 1-2x per month.
3. **Decision Frameworks** — Structured architecture decision records with Mermaid diagrams. Published monthly.
4. **Meta / Career** — Posts about engineering leadership, leverage, and career development. Occasional.

## Success Metrics
- Time from "idea in Obsidian" to "live on the web" < 2 minutes
- Site loads in < 1 second on mobile
- Zero ongoing infrastructure cost (GitHub Pages free tier)
- Supports full-text search, tags, backlinks, and RSS out of the box
