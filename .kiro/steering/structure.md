# Project Structure

```
my-blog/
├── .kiro/                          # Kiro spec files (this directory)
│   ├── specs/
│   │   └── personal-blog/
│   │       ├── requirements.md
│   │       ├── design.md
│   │       └── tasks.md
│   └── steering/
│       ├── product.md
│       ├── tech.md
│       └── structure.md
├── .github/
│   └── workflows/
│       └── deploy.yml              # GitHub Actions deploy workflow (Quartz default)
├── content/                        # ALL blog content lives here
│   ├── index.md                    # Homepage / landing page content
│   ├── about.md                    # About page
│   ├── blog-ideas.md               # Idea capture note (draft: true, never published)
│   ├── why-im-starting-this-blog.md # First post
│   ├── til-leverage-hierarchy-for-engineering.md  # Sample TIL
│   └── tags/                       # Tag index pages (auto-generated by Quartz)
├── templates/                      # Obsidian Templater templates (not published)
│   ├── til-template.md
│   ├── deep-dive-template.md
│   └── decision-framework-template.md
├── quartz/                         # Quartz source (do not modify unless necessary)
│   ├── styles/
│   │   └── custom.scss             # Custom CSS overrides — this IS our design layer
│   ├── components/                 # Quartz Preact components
│   └── plugins/                    # Quartz plugin system
├── static/                         # Global static assets (favicon, og-image, etc.)
│   ├── favicon.ico
│   └── og-image.png
├── quartz.config.ts                # Primary configuration file
├── quartz.layout.ts                # Layout configuration (header, sidebar, footer)
├── package.json
└── tsconfig.json
```

## Key Conventions
- `content/` is the ONLY directory authors touch day-to-day
- `templates/` is excluded from Quartz build via `ignorePatterns` config
- Files with `draft: true` in frontmatter are excluded from the published site
- Quartz auto-generates: tag pages, backlink sections, graph view, search index, RSS feed, sitemap
- Custom styling lives exclusively in `quartz/styles/custom.scss`
- Configuration changes go in `quartz.config.ts` and `quartz.layout.ts` — never modify Quartz internals
